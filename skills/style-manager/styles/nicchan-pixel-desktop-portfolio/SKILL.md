# Nicchan Pixel Desktop Portfolio

This child skill recreates the style family of an accessible pixel-art desktop portfolio, without copying the source site's content or exact layout.

Default page type: independent developer/designer portfolio.

## Read First

1. `DESIGN.md` for tokens and component rules.
2. `references/source-observations.md` for provenance and CSS extraction.
3. `references/code-patterns/` for reusable HTML/CSS fragments.
4. `preview.html` for a fast token scan.
5. `examples/index.html` for a fresh style-transfer reference site.

## Preserve

- Purple pixel-art wallpaper as the full viewport field.
- Fixed left desktop-icon navigation.
- Bottom taskbar with open-window buttons or settings.
- Window components with gradient header, title, and square controls.
- W95fa/Sysfont/Departure Mono style hierarchy.
- 3px square borders, inset bevels, and 1px pressed translation.
- Strong accessibility cues: focus outlines, reduced-motion respect, clear labels.

## Avoid

- Smooth SaaS cards, rounded corners, glass blur, soft shadows.
- Generic neon-retro or arcade UI.
- Copying Nic Chan's portrait, client logos, posts, or information architecture.
- Hiding the OS metaphor on mobile.

When adapting into another framework, keep the visual logic first and translate the component mechanics second.
