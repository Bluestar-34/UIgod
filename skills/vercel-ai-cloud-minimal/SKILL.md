# Vercel AI Cloud Minimal

This child skill captures a Vercel-inspired developer infrastructure style derived from `https://vercel.com/`.

It is designed for web work that should feel like a precise, high-performance technical platform rather than a generic dark SaaS page.

Default representative use cases: developer tool homepages, AI infrastructure sites, cloud deployment products, engineering portfolios, and technical product launches. Generate a fresh information architecture for the user's requested site type while preserving this style.

## Use When

Use this skill for:

- developer tool homepages
- AI infrastructure platforms
- cloud deployment products
- engineering portfolios
- technical product launches
- premium black-and-white SaaS surfaces

## Style Priority

Preserve:

- black canvas
- strict hairline grid
- centered, concise hero
- white pill primary CTA
- triangle geometry
- spectral ray/light effects
- product cards as infrastructure modules
- Geist-like sans and mono pairing

Framework choice comes after the style logic is understood.

Use `DESIGN.md` as the single design truth source. Treat this file as the operational wrapper around it.

When consulting examples, study visual decisions, spacing, component behavior, and interaction density. Do not copy page structure unless the user explicitly asks for that exact structure.

## Expected Output

A strong result should feel like a launch surface for fast technical systems:

- black-and-white first
- exact grid alignment
- luminous but restrained hero graphic
- compact developer UI details
- no decorative fluff

## Guardrails

- Do not turn the style into neon cyberpunk.
- Do not add pastel cards, soft blobs, or lifestyle imagery.
- Do not clone the Vercel homepage as the default output.
- Keep color as emitted light or status accents.
- Keep buttons compact and pill-shaped.
- Keep cards modular, bordered, and technical.
