# UIgod MOC

## Goal

UIgod is a Superpowers-style flat skills repository with a managed style library.

The default visible skills are [skills/uigod](skills/uigod/) and [skills/style-manager](skills/style-manager/). UIgod reads user-supplied design references and generates managed style packages under `skills/style-manager/styles/<style-name>/`.

## Install Boundary

- Default install: `skills/uigod/` and `skills/style-manager/`
- Generated library entries: `skills/style-manager/styles/<style-name>/`
- Optional standalone export only on request: any specific managed style

## Workspace

- `skills/uigod/`: default installed meta-skill
- `skills/style-manager/`: default installed style selector
- `skills/style-manager/styles/<style-name>/`: managed style packages generated by UIgod
- `skills/uigod/templates/`: starter resources, not installable skills
- `inputs/`: captured evidence and screenshots
- `outputs/`: scratch generated outputs
- `agents/`: optional agent configuration

## Current Focus

1. Keep only coordinator skills in the visible `skills/` namespace.
2. Install `skills/uigod/` and `skills/style-manager/` by default.
3. Store concrete styles under `skills/style-manager/styles/`.
4. Keep every managed style led by `DESIGN.md` with nested `SKILL.md` as the short operational wrapper.
